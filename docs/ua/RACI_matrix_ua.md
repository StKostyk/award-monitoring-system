# RACI Матриця: Ролі та Обов'язки

## Легенда
- **R = Відповідальний:** Виконує роботу для завершення завдання
- **A = Підзвітний:** В кінцевому підсумку відповідає за завершення та підпис
- **C = Консультований:** Надає вхідні дані та експертизу (двостороння комунікація)
- **I = Поінформований:** Тримається в курсі прогресу (одностороння комунікація)

| **Налаштувати політики зберігання даних** | C | I | I | I | C | A | R |
| **Проектувати архітектуру системи** | C | I | C | C | A | C | R |
| **Впровадити засоби контролю безпеки** | I | I | I | C | A | C | R |
| **Розгорнути функції відповідності GDPR** | C | I | I | I | C | A | R |
| **Провести тестування прийняття користувачів** | I | A | R | I | I | I | C |
| **Схвалити продакшн розгортання** | A | I | C | C | C | C | R |

## 5. Шляхи та Процедури Ескалації

### Технічний Шлях Ескалації
```
Рівень 1: Команда Розробки
    ↓ (Якщо не вирішено протягом 24 годин)
Рівень 2: Системні Оператори + Команда InfoSec  
    ↓ (Якщо не вирішено протягом 48 годин)
Рівень 3: Декан (Проф. Мартинюк)
    ↓ (Якщо не вирішено протягом 72 годин)
Рівень 4: Ректор (Проф. Білоскурський)
```

### Бізнес Шлях Ескалації
```
Рівень 1: Секретар Факультету
    ↓ (Якщо конфлікт політики або незвичайний випадок)
Рівень 2: Декан (Проф. Мартинюк)
    ↓ (Якщо вплив на інституційну політику)
Рівень 3: Секретар Ректора (Наталія Якубовська)
    ↓ (Якщо потрібне виконавче рішення)
Рівень 4: Ректор (Проф. Білоскурський)
```

### Шлях Ескалації Комплаєнсу
```
Рівень 1: Офіцер GDPR
    ↓ (Якщо потрібна юридична інтерпретація)
Рівень 2: Декан + Команда InfoSec
    ↓ (Якщо конфлікт інституційної політики)
Рівень 3: Ректор (Проф. Білоскурський)
    ↓ (Якщо потрібна зовнішня юридична консультація)
Рівень 4: Зовнішня Юридична Консультація
```

## 6. Вимоги до Комунікації

### Матриця Комунікації Рішень
| Рівень Рішення | Метод Комунікації | Стейкхолдери для Сповіщення | Часові Рамки |
|----------------|-------------------|------------------------------|--------------|
| **Стратегічний** | Формальний email + зустріч | Всі стейкхолдери | Протягом 24 годин |
| **Тактичний** | Email сповіщення | Відповідні стейкхолдери | Протягом 48 годин |
| **Технічний** | Технічна документація | Технічні стейкхолдери | Протягом 72 годин |
| **Операційний** | Системне сповіщення | Кінцеві користувачі | Реальний час/наступний робочий день |

### Робочі Процеси Схвалення
- **Зміни Політики:** Офіцер GDPR → Декан → Ректор (послідовно)
- **Технічні Зміни:** Огляд InfoSec → Розробка → Декан (паралельний огляд)
- **Зміни Бюджету:** Розробка → Декан → Ректор (послідовно)
- **Екстрені Зміни:** Паралельне сповіщення, ретроспективне схвалення
