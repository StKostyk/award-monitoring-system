# ADR-002: Вибір версії Java

**Статус**: Прийнято  
**Дата**: 2025-08-20  
**Автор**: Стефан Костик  
**Зацікавлені сторони**: Архітектор проекту, Команда розробки

---

## Контекст

Система моніторингу та відстеження нагород потребує сучасної версії Java, яка забезпечує покращення продуктивності, оновлення безпеки та мовні функції, зберігаючи при цьому корпоративну підтримку та сумісність екосистеми.

### Передумови
- Потреба в останній LTS версії для довгострокової підтримки та стабільності
- Покращення продуктивності для кращої чутливості додатка
- Сучасні мовні функції для підвищення продуктивності розробника
- Вимоги сумісності ентерпрайз екосистеми

---

## Рішення

**OpenJDK 21 (LTS)** було обрано як версія Java для Системи моніторингу та відстеження нагород.

### Обраний підхід
- **JDK**: OpenJDK 21 LTS (дистрибуція Eclipse Temurin)
- **Термін підтримки**: LTS підтримка до вересня 2031
- **Рівень мови**: Функції Java 21 увімкнено
- **Середовище виконання**: OpenJDK 21 для розробки та продакшену

### Обґрунтування
- **LTS підтримка**: Остання LTS версія з розширеним життєвим циклом підтримки
- **Продуктивність**: Значні покращення продуктивності порівняно з Java 17
- **Сучасні функції**: Pattern matching, sealed класи, віртуальні потоки
- **Сумісність Spring Boot**: Повна підтримка в Spring Boot 3.2+
- **Корпоративне прийняття**: Зростаюче корпоративне прийняття Java 21

---

## Наслідки

### Позитивні наслідки
- **Продуктивність**: Кращий час запуску додатка та продуктивність виконання
- **Продуктивність розробника**: Сучасні мовні функції зменшують шаблонний код
- **Довгострокова підтримка**: LTS версія забезпечує стабільність та оновлення безпеки
- **Сумісність екосистеми**: Повна сумісність з обраним технічним стеком

### Негативні наслідки
- **Крива навчання**: Команді потрібно вивчити нові функції Java 21
- **Інструментарій**: Деякі інструменти можуть потребувати оновлень для повної підтримки Java 21

---

## Розглянуті альтернативи

### Альтернатива 1: OpenJDK 17 LTS
- **Переваги**: Ширше прийняття, доведена стабільність, ширша підтримка інструментів
- **Недоліки**: Відсутність останніх покращень продуктивності та мовних функцій
- **Причина відхилення**: Java 21 пропонує значні переваги для нових проектів

### Альтернатива 2: OpenJDK 22/23
- **Переваги**: Останні функції та покращення
- **Недоліки**: Не LTS версії, коротший життєвий цикл підтримки
- **Причина відхилення**: Корпоративні проекти потребують стабільності LTS

---

## Примітки з реалізації

### Технічні вимоги
- **Мінімальна версія**: OpenJDK 21.0.1
- **Дистрибуція**: Рекомендується Eclipse Temurin
- **Конфігурація збірки**: Maven compiler plugin налаштований для Java 21
- **Підтримка IDE**: IntelliJ IDEA 2023.3+ або VS Code з Java розширеннями

### Кроки реалізації
1. Встановити OpenJDK 21 на середовища розробки та розгортання
2. Налаштувати Maven для використання рівня мови Java 21
3. Оновити CI/CD pipeline для використання середовища виконання Java 21
4. Перевірити сумісність всіх залежностей з Java 21

---

## Метрики успіху

- **Сумісність**: 100% сумісність з обраним технічним стеком
- **Продуктивність**: Встановлені базові вимірювання продуктивності
- **Успіх збірки**: Чисті збірки без проблем версії Java
