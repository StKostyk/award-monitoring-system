# ADR-014: Вибір бібліотеки UI компонентів

**Статус**: Прийнято  
**Дата**: 2025-08-21  
**Автор**: Стефан Костик  
**Зацікавлені сторони**: Архітектор проекту, Frontend команда, UX команда

---

## Контекст

Система моніторингу та відстеження нагород потребує комплексної бібліотеки UI компонентів, що забезпечує консистентні, доступні та професійно виглядаючі компоненти. Бібліотека повинна безшовно інтегруватися з Angular та підтримувати theming, інтернаціоналізацію та вимоги доступності.

### Передумови
- Потреба в консистентній дизайн системі по всьому додатку
- Вимоги доступності (відповідність WCAG AA)
- Професійний вигляд, підходящий для корпоративних середовищ
- Підтримка тем та кастомізації
- Інтеграція з Angular екосистемою

### Припущення
- Бібліотеки з підтримкою Google забезпечують кращу довгострокову підтримку та інтеграцію
- Принципи Material Design узгоджуються з сучасними корпоративними очікуваннями UI
- Комплексна бібліотека компонентів зменшує час розробки
- Вбудовані функції доступності є важливими для відповідності

---

## Рішення

**Angular Material** було обрано як бібліотека UI компонентів для Системи моніторингу та відстеження нагород.

### Обраний підхід
- **Бібліотека компонентів**: Angular Material (остання стабільна версія)
- **Дизайн система**: Google Material Design 3
- **Theming**: Користувацька тема на основі Material Design токенів
- **Іконки**: Material Icons з додаванням користувацьких іконок за потребою

### Обґрунтування
- **Підтримка Google**: Офіційна підтримка Google забезпечує довгострокову життєздатність та інтеграцію
- **Функції доступності**: Вбудована відповідність WCAG AA та функції доступності
- **Консистентний дизайн**: Material Design забезпечує цілісний та професійний вигляд
- **Angular інтеграція**: Спеціально створений для Angular з відмінною підтримкою TypeScript
- **Комплексні компоненти**: Широкий спектр компонентів, що покривають більшість корпоративних випадків використання

## Наслідки

### Позитивні наслідки
- **Швидкість розробки**: Готові компоненти прискорюють розробку
- **Доступність**: Вбудовані функції доступності зменшують зусилля з відповідності
- **Консистентність**: Уніфікована дизайн мова по всьому додатку
- **Обслуговування**: Підтримка Google забезпечує регулярні оновлення та виправлення багів
- **Документація**: Відмінна документація та приклади

### Негативні наслідки
- **Обмеження дизайну**: Material Design може обмежувати унікальні дизайнерські рішення
- **Розмір bundle**: Додає до розміру bundle додатка
- **Крива навчання**: Розробники повинні вивчити принципи Material Design

### Нейтральні наслідки
- **Складність тематизації**: Просунута тематизація вимагає розуміння Material Design токенів
- **Кастомізація**: Деякі компоненти можуть потребувати додаткової кастомізації для специфічних потреб

## Розглянуті альтернативи

### Альтернатива 1: PrimeNG
- **Переваги**: Багатий набір компонентів, обширні функції, хороша документація
- **Недоліки**: Менш Angular-нативний вигляд, додаткове ліцензування для просунутих функцій
- **Причина відхилення**: Angular Material забезпечує кращу інтеграцію екосистеми

### Альтернатива 2: Nebular
- **Переваги**: Eva Design System, хороші можливості тематизації, орієнтованість на Angular
- **Недоліки**: Менша спільнота, менше корпоративного прийняття
- **Причина відхилення**: Нижче прийняття та менша екосистема

### Альтернатива 3: Користувацька бібліотека компонентів
- **Переваги**: Повний контроль над дизайном та функціональністю
- **Недоліки**: Значний час розробки, накладні витрати на обслуговування, відсутність вбудованої доступності
- **Причина відхилення**: Не практично для термінів одного розробника

## Примітки з реалізації

### Технічні вимоги
- **Версія Angular Material**: Остання сумісна з Angular 20+
- **CDK**: Angular Component Development Kit для просунутих поведінок компонентів
- **Анімації**: Angular Animations для Material компонентів
- **Іконки**: Material Icons та користувацькі SVG іконки

### Встановлення та налаштування
```bash
ng add @angular/material
npm install @angular/cdk
npm install @angular/animations
```

### Конфігурація теми
```scss
@use '@angular/material' as mat;

$primary-palette: mat.define-palette(mat.$blue-palette);
$accent-palette: mat.define-palette(mat.$amber-palette);
$warn-palette: mat.define-palette(mat.$red-palette);

$theme: mat.define-light-theme((
  color: (
    primary: $primary-palette,
    accent: $accent-palette,
    warn: $warn-palette,
  )
));

@include mat.all-component-themes($theme);
```

### Використання основних компонентів
- **Навігація**: Toolbar, Sidenav, Menu
- **Макет**: Card, Expansion Panel, Tabs, Stepper
- **Форми**: Input, Select, Checkbox, Radio, Datepicker
- **Відображення даних**: Table, List, Paginator, Sort
- **Зворотній зв'язок**: Dialog, Snackbar, Progress Bar, Spinner

## Відповідність та якість

### Наслідки для доступності
- **Відповідність WCAG AA**: Вбудовані функції доступності у всіх компонентах
- **Підтримка screen reader**: Правильні ARIA мітки та живі регіони
- **Навігація клавіатурою**: Повна підтримка навігації клавіатурою
- **Високий контраст**: Підтримка тем високого контрасту

### Вплив на продуктивність
- **Tree Shaking**: Імпорт лише необхідних компонентів для мінімізації розміру бандла
- **OnPush стратегія**: Сумісність з Angular OnPush виявленням змін
- **Віртуальна прокрутка**: Доступна для великих наборів даних у таблицях та списках

### Супроводжуваність
- **Консистентний API**: Стандартизований API компонентів для всіх Material компонентів
- **Підтримка TypeScript**: Повні TypeScript визначення та IntelliSense
- **Тестування**: Angular Testing Utilities для Material компонентів

## Метрики успіху

### Ключові показники ефективності
- **Оцінка доступності**: 100% відповідність axe-core
- **Швидкість розробки**: 50% скорочення часу розробки UI
- **Вплив на розмір bundle**: < 200KB додаткового розміру bundle

### Моніторинг та сповіщення
- **Тестування доступності**: Автоматизоване тестування доступності в CI/CD
- **Моніторинг розміру bundle**: Відстеження впливу Material компонентів на розмір bundle
- **Користувацький досвід**: Моніторинг шаблонів взаємодії користувачів з Material компонентами
