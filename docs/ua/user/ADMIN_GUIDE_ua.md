## Огляд ролей адміністратора

### Ієрархія ролей

| Роль | Область | Ключові дозволи |
|------|---------|-----------------|
| **Системний адмін** | Глобальна | Повний доступ, керування користувачами, конфігурація |
| **Офіцер GDPR** | Глобальна | Запити на доступ до даних, видалення, журнали аудиту |
| **Ректор** | Університет | Фінальне затвердження, всі звіти |
| **Декан** | Факультет | Затвердження факультету, звіти факультету |
| **Секретар факультету** | Факультет | Перегляд нагород, допомога користувачам |
| **Секретар кафедри** | Кафедра | Первинний перегляд, перевірка документів |

### Доступ до адміністративних функцій

1. Увійдіть з обліковим записом адміністратора
2. Натисніть **"Адміністрування"** у навігації
3. Виберіть потрібний адміністративний розділ

## Керування користувачами

### Перегляд користувачів

```
Адміністрування → Користувачі → Список користувачів
```

| Фільтр | Опції |
|--------|-------|
| Роль | Всі / Конкретна роль |
| Організація | Університет / Факультет / Кафедра |
| Статус | Активний / Неактивний / Призупинений |
| Пошук | Ім'я, email |

### Створення користувачів

1. Перейдіть до **Адміністрування → Користувачі → Додати користувача**
2. Заповніть обов'язкові поля:
   - Email (потрібен домен університету)
   - Ім'я/Прізвище
   - Роль
   - Призначення організації

3. Користувач отримає email активації

### Зміна ролей користувачів

```
Користувачі → Вибрати користувача → Редагувати → Змінити роль → Зберегти
```

**Правила зміни ролей:**
- Не можна самостійно призначити вищі привілеї
- Зміни ролей записуються в журнал аудиту
- Користувач повідомляється про зміни ролей

### Деактивація користувачів

1. Виберіть користувача зі списку
2. Натисніть **"Деактивувати"**
3. Підтвердіть дію
4. Користувач негайно втрачає доступ
5. Нагороди залишаються, але призначаються наступнику

### Масові операції з користувачами

```
Адміністрування → Користувачі → Масові дії
```

- Імпорт користувачів через CSV
- Експорт списку користувачів
- Масові зміни статусу
- Масове призначення організації

**Формат CSV імпорту:**
```csv
email,firstName,lastName,role,organization
ivan@uni.edu.ua,Іван,Петренко,EMPLOYEE,Кафедра КН
maria@uni.edu.ua,Марія,Коваль,FACULTY_SECRETARY,Факультет ІТ
```

## Керування категоріями нагород

### Керування категоріями

```
Адміністрування → Нагороди → Категорії
```

### Створення категорій

1. Натисніть **"+ Додати категорію"**
2. Налаштуйте:
   - Назва (українською та англійською)
   - Рівень (Міжнародний/Національний/Регіональний/Локальний)
   - Опис
   - Необхідні документи
   - Робочий процес затвердження

### Налаштування категорій

| Налаштування | Опис |
|--------------|------|
| **Активна** | Доступна для нових подань |
| **Потребує верифікації** | Додатковий крок перевірки |
| **Авто-публікація** | Публікувати одразу після затвердження |
| **Шаблон сповіщення** | Власний шаблон email |

### Архівування категорій

- Архівовані категорії не можуть приймати нові нагороди
- Існуючі нагороди в архівованих категоріях залишаються видимими
- Можна відновити за потреби

## Налаштування робочих процесів

### Робочий процес за замовчуванням

```
Працівник подає
     ↓
Перегляд кафедри (Секретар)
     ↓
Перегляд факультету (Декан)
     ↓
Перегляд університету (Офіс ректора)
     ↓
Фінальне затвердження (Ректор)
     ↓
Опубліковано
```

### Налаштування робочих процесів

```
Адміністрування → Робочі процеси → Редагувати
```

**Конфігуровані опції:**
- Пропуск рівнів для певних категорій
- Паралельні затвердження
- Авто-затвердження для конкретних типів нагород
- Вимоги дедлайнів на рівень

### Правила призначення рецензентів

| Тип правила | Опис |
|-------------|------|
| Автоматичне | Призначається власнику ролі в організації |
| Ручне | Адміністратор призначає конкретного рецензента |
| Ротаційне | Розподіляється серед відповідних рецензентів |
| Збалансоване | Призначається рецензенту з найменше очікуваних |

### Налаштування дедлайнів

```yaml
Дедлайни перегляду:
  Кафедра: 3 робочих дні
  Факультет: 5 робочих днів
  Університет: 7 робочих днів
  
Ескалація:
  Попередження: за 1 день до дедлайну
  Ескалація: в день дедлайну
  Авто-передача: через 2 дні після дедлайну
```

## Звіти та аналітика

### Доступні звіти

| Звіт | Аудиторія | Зміст |
|------|-----------|-------|
| Підсумок нагород | Всі адміни | Розподіл статусів, тренди |
| Час обробки | Керівники | Середній час перегляду, вузькі місця |
| Активність користувачів | Системний адмін | Вхід, подання, перегляди |
| Відповідність | Офіцер GDPR | Доступ до даних, видалення |
| Звіт факультету | Декани | Статистика факультету |

### Генерація звітів

1. Перейдіть до **Адміністрування → Звіти**
2. Виберіть тип звіту
3. Встановіть параметри (діапазон дат, фільтри)
4. Виберіть формат (PDF, Excel, CSV)
5. Згенеруйте та завантажте

### Заплановані звіти

Налаштуйте автоматичну генерацію звітів:

```
Адміністрування → Звіти → Заплановані
```

| Налаштування | Опції |
|--------------|-------|
| Частота | Щодня / Щотижня / Щомісяця |
| Отримувачі | Email адреси |
| Формат | PDF / Excel |
| Час | Бажаний час доставки |

## Конфігурація системи

### Загальні налаштування

```
Адміністрування → Налаштування → Загальні
```

| Налаштування | Опис |
|--------------|------|
| Назва системи | Відображається в UI та emails |
| Email підтримки | Контакт підтримки користувачів |
| Таймаут сесії | Час авто-виходу (хвилини) |
| Ліміт розміру файлу | Макс. розмір завантаження (МБ) |
| Дозволені типи файлів | PDF, JPEG, PNG |

### Налаштування Email

```
Адміністрування → Налаштування → Email
```

- Налаштування SMTP сервера
- Шаблони email (за типом сповіщення)
- Ім'я та адреса відправника
- Функціонал тестового email

### Налаштування безпеки

```
Адміністрування → Налаштування → Безпека
```

| Налаштування | Рекомендовано |
|--------------|---------------|
| Політика паролів | Мін. 8 символів, різний регістр, число, символ |
| MFA | Обов'язково для адмінів, опціонально для користувачів |
| Тривалість сесії | 8 годин |
| Блокування після невдалих входів | 5 спроб, 30 хв блокування |
| IP білий список | Опціонально для адмін доступу |

### Режим обслуговування

Увімкніть під час оновлень або аварій:

```
Адміністрування → Налаштування → Режим обслуговування
```

- Відображає повідомлення про обслуговування користувачам
- Дозволяє тільки адмін доступ
- Ставить в чергу вхідні подання
- Авто-вимкнення після встановленого часу

## Аудит та відповідність

### Журнали аудиту

```
Адміністрування → Аудит → Журнали
```

**Записані події:**
- Вхід/вихід користувачів
- Подання нагород та зміни статусу
- Завантаження/видалення документів
- Адміністративні дії
- Зміни конфігурації
- Експорт даних

### Пошук у журналах аудиту

| Фільтр | Приклад |
|--------|---------|
| Користувач | `user:ivan@uni.edu.ua` |
| Дія | `action:DELETE` |
| Ресурс | `resource:award` |
| Дата | `date:2026-01-01..2026-01-31` |
| IP | `ip:192.168.1.*` |

### Відповідність GDPR

#### Запити суб'єктів даних

```
Адміністрування → GDPR → Запити на дані
```

**Типи запитів:**
- Доступ (експорт даних користувача)
- Виправлення (коригування даних)
- Видалення (видалення даних)
- Портативність (експорт у стандартному форматі)

**Обробка запитів:**
1. Перевірити особу заявника
2. Переглянути тип запиту
3. Згенерувати звіт або виконати дію
4. Задокументувати завершення
5. Повідомити користувача

#### Збереження даних

```
Адміністрування → GDPR → Збереження
```

| Тип даних | Збереження | Після закінчення |
|-----------|------------|------------------|
| Активні нагороди | Безстроково | Н/Д |
| Відхилені нагороди | 3 роки | Анонімізація |
| Журнали аудиту | 7 років | Видалення |
| Неактивні облікові записи | 2 роки | Анонімізація |

## Усунення несправностей

### Типові проблеми адміністратора

**Проблема: Користувач не може увійти**
1. Перевірте статус користувача (активний/неактивний)
2. Перевірте правильність email
3. Перевірте блокування облікового запису
4. Скиньте пароль за потреби

**Проблема: Робочий процес застряг**
1. Перевірте поточний рівень перегляду
2. Перевірте призначення рецензента
3. Перевірте перевищення дедлайну
4. Призначте рецензента вручну за потреби

**Проблема: Звіти не генеруються**
1. Перевірте валідність діапазону дат
2. Перевірте наявність даних для фільтрів
3. Спробуйте менший діапазон дат
4. Перевірте журнали сервера на помилки

### Інструменти адміністратора

```
Адміністрування → Інструменти
```

| Інструмент | Призначення |
|------------|-------------|
| Очистка кешу | Очистити системні кеші |
| Переіндексація пошуку | Перебудувати пошуковий індекс |
| Перевірка стану | Статус здоров'я системи |
| Статус резервного копіювання | Інформація про останнє резервне копіювання |

### Отримання технічної допомоги

Для проблем поза областю адміністратора:
1. Перевірте [Посібник з усунення несправностей](../operations/TROUBLESHOOTING_ua.md)
2. Зверніться до IT підтримки: it-support@university.edu.ua
3. Для термінових проблем: див. [Реагування на інциденти](../operations/runbooks/INCIDENT_RESPONSE_RUNBOOK_ua.md)

## Найкращі практики

### Керування користувачами
- Регулярний аудит ролей та доступу користувачів
- Швидке видалення неактивних користувачів

- Щорічне тестування відновлення резервних копій
- Документування рішень щодо збереження даних

### Безпека
- Увімкнення MFA для всіх адмін облікових записів
- Щотижневий перегляд журналів аудиту
- Щоквартальна зміна адмін паролів
- Тестування процедур реагування на інциденти
