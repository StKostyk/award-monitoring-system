## 6. Відстеження прогресу

### 6.1 Метрики для відстеження

| **Метрика** | **Ціль** | **Вимірювання** |
|-------------|----------|-----------------|
| Velocity | 15-20 балів/спринт | Завершені story points |
| Успіх мети спринту | >80% | Досягнуті цілі за спринт |
| Рівень витоку багів | <5% | Баги, знайдені після спринту |
| Покриття коду | >85% | Звіти SonarQube |
| Технічний борг | <5% | Коефіцієнт боргу SonarQube |

### 6.2 Відстеження Burndown

Відстежуйте щодня в спринті:
```
День 1  │████████████████████│ 15 балів залишилось
День 3  │███████████████     │ 12 балів залишилось
День 5  │███████████         │  9 балів залишилось
День 7  │███████             │  6 балів залишилось
День 10 │███                 │  3 балів залишилось
День 14 │                    │  0 балів залишилось ✓
```

### 6.3 Кумулятивний потік

Відстежуйте щотижня між спринтами для виявлення вузьких місць:
- Тренд розміру беклогу
- Накопичення WIP
- Темп завершення
